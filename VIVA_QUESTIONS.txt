# AI Podcast ScriptWriter: Viva Questions & Concepts

### 1. General AI & Agents
Q: What makes this "Agentic AI" rather than just a Chatbot?
A: A chatbot is one-shot (input -> output). This system is "Agentic" because it uses multi-step reasoning, autonomous tool use (Tavily Search), and a self-correcting feedback loop (LLM-as-Judge). It breaks a goal into sub-tasks and executes them independently.

Q: Why use CrewAI instead of a simple LangChain prompt?
A: CrewAI provides a framework for role-playing, where agents have specific backstories and goals. It handles "Process Management" (Sequential vs. Hierarchical) and allows agents to pass context to one another.

### 2. Architecture & Challenges
Q: How did you solve the LLM output limit (truncation) issue?
A: I implemented "Segmented Writing & Hard-Concatenation." The task is divided into 6 standalone segments. Each segment is small enough to fit in the LLM's output buffer (~500 words). Python then joins these strings together, bypassing the 1000-word "ceiling" most LLMs have.

Q: Explain your "Hot Fallback" logic.
A: The system attempts to use Gemini 2.0 (Cloud) first for speed. If it detects a 404 (model not found), 429 (rate limit), or 400 (API error), it automatically catches the exception and restarts the task using the local Ollama (Llama 3.2) engine on the user's GPU.

### 3. Tools & Data
Q: What is Tavily, and why use it over Google Search?
A: Tavily is an AI-native search engine. Unlike standard Google Search which returns HTML junk, Tavily returns "AI-ready" clean content and summaries, reducing the token noise and improving the Researcher agent's accuracy.

Q: What is "LLM-as-Judge"?
A: It is a design pattern where a separate LLM agent (The Critic) evaluates the output of a previous agent based on a rubric (word count, flow, accuracy). This creates a "closed-loop" system where the AI quality-checks itself.

### 4. Technical Hardware
Q: Why did you prioritize Llama 3.2 3B for local use?
A: Because it has a tiny memory footprint (~2GB), which fits perfectly on the user's 8GB RAM / RTX 3050 setup, while still being one of the best small models for tool-calling/function-calling.
